- 直接修改当前工作区文件完成任务
- 如果任务明确要求使用仓库现有命令、脚手架或规范来生成制品，优先遵守该流程，而不是自行发明文件结构

- 不要创建 git commit

返回内容应覆盖：

- 这轮做了什么
- 改了哪些文件
- 做了哪些验证
- 还剩哪些问题需要人类确认；如果没有，则返回空数组
