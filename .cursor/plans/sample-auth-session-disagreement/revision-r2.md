# 统一登录态策略计划

## 任务理解

需要为 Web 与移动端设计一套统一登录态管理方案，并输出开发计划。

## 目标

- 统一身份校验入口
- 减少现有登录态实现分叉
- 降低迁移期间的用户登录中断风险

## 范围

- 覆盖 Web 与移动端
- 覆盖登录、续期和登出
- 不处理第三方单点登录

## 假设

- Web 继续使用 Cookie Session
- 移动端可在登录后换取短期 session token，并映射到同一份服务端 Session 记录

## 风险

- 移动端 token 与服务端 Session 的续期边界容易不一致
- 迁移期可能出现重复登录

## 建议方案

服务端保留统一 Session 存储。Web 继续使用 Cookie 承载 session ID。移动端在登录后换取短期 session token，并由网关在请求入口把 token 解析为同一份服务端 Session。这样服务端仍能统一管理登录态，只是在客户端载体上区分 Web 与移动端。

## 验证方式

- 登录成功率不低于现网基线
- Session 续期成功率达到 99.9%
- 强制重新登录率不高于现网基线的 0.2%

## 开放问题

- 移动端是否需要离线续期能力

## 执行步骤
1. 抽象统一 Session 存储接口。
2. 保持 Web 的 Cookie Session 路径不变。
3. 为移动端新增短期 session token 换取与校验逻辑。
4. 在迁移期观测登录与续期指标。
