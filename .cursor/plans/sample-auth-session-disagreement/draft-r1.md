# 统一登录态策略计划

## 任务理解

需要为 Web 与移动端设计一套统一登录态管理方案，并输出开发计划。

## 目标

- 统一身份校验入口
- 减少现有登录态实现分叉
- 降低迁移期间的用户登录中断风险

## 范围

- 覆盖 Web 与移动端
- 覆盖登录、续期和登出
- 不处理第三方单点登录

## 假设

- Web 与移动端都可以直接复用同一套基于 Cookie 的 Session 模式

## 风险

- 迁移期可能出现重复登录
- 会话续期策略不一致可能引发投诉

## 建议方案

统一采用服务端 Session 存储，所有客户端都通过同一套 Cookie 会话续期。移动端通过内嵌 WebView 同步 Cookie，使服务端无需区分客户端类型。

## 验证方式

- 验证 Web 与移动端都能完成登录和登出
- 验证迁移后投诉量没有明显上升

## 开放问题

- 是否需要支持离线场景

## 执行步骤
1. 抽象统一 Session 接口。
2. 让 Web 与移动端都接入同一套 Cookie Session。
3. 上线后观察投诉量。
